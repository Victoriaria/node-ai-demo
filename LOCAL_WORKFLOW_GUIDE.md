# 🔧 NODE AI 本地工作流引擎指南

## 概述

本地工作流引擎完全替代了 Coze Workflow，实现了：
- ✅ 完全本地化，无需依赖 Coze API
- ✅ 响应速度快（<100ms）
- ✅ 100% 可控和可定制
- ✅ 集成反欺诈案例库
- ✅ 智能关键词匹配
- ✅ 多级风险评估

---

## 架构设计

### 工作流节点

```
开始 → 问答节点 → 条件分支 → 大模型分析 → 结束
  ↓        ↓          ↓           ↓         ↓
 输入   关键词提取   风险分流   深度分析   生成报告
```

### 核心组件

1. **workflowEngine.ts** - 工作流引擎
   - 节点编排
   - 上下文管理
   - 决策逻辑

2. **fraudCaseDatabase.ts** - 反欺诈案例库
   - 案例匹配
   - 模式识别
   - 风险评分

3. **API Route** - 对外接口
   - 接收用户输入
   - 调用工作流引擎
   - 返回审计结果

## 工作流详解

### 节点1: 问答节点（QA Node）

**功能**：关键词提取与初步分析

**处理逻辑**：
1. 提取输入中的关键词
2. 计算关键词权重
   - 高风险关键词：权重 15
   - 中风险关键词：权重 8
   - 低风险关键词：权重 3
3. 匹配反欺诈案例库
4. 计算初步风险评分

**关键词库**：
```typescript
CRITICAL: ['JD', '换汇', '拆单', '代收款', '洗钱', '诈骗', ...]
HIGH: ['跨境', '大额', '转账', '频繁', '异常', ...]
MEDIUM: ['支付', '汇款', '收款', '账户', ...]
```

**输出**：
- 识别的关键词列表
- 匹配的案例ID
- 初步风险评分

---

### 节点2: 条件分支（Condition Node）

**功能**：根据风险评分分流

**分支逻辑**：
```typescript
if (score >= 80) → 高风险分支
else if (score >= 60) → 中风险分支
else → 低风险分支
```

**输出**：
- 风险等级（CRITICAL / HIGH / LOW）
- 审计模式（AUDIT / NORMAL）
- 分支路径

---

### 节点3: 大模型分析（LLM Node）

**功能**：深度风险分析与决策

#### 3a. 高风险分支

**处理逻辑**：
1. 检查是否缺少关键信息
   - 缺少 → 返回 INFO_REQUIRED
   - 完整 → 继续分析
2. 评估风险等级
   - score >= 90 → HALT（熔断）
   - 60 <= score < 90 → WARN（警告）
3. 生成详细原因

**决策规则**：
```typescript
score >= 90 → HALT（立即拦截）
score >= 60 → WARN（人工复核）
```

#### 3b. 中风险分支

**处理逻辑**：
1. 分析匹配的案例
2. 生成风险提示
3. 建议加强监控

**决策**：WARN（警告）

#### 3c. 低风险分支

**处理逻辑**：
1. 确认无明显风险
2. 允许交易

**决策**：ALLOW（允许）

---

### 节点4: 结束节点（End Node）

**功能**：生成审计报告

**输出格式**：

## 使用方式

### 1. 环境变量配置

在 `.env.local` 中设置：

```env
# 启用本地工作流引擎
USE_LOCAL_WORKFLOW=true

# 关闭 Coze API
USE_MOCK_DATA=false
```

### 2. 启动开发服务器

```bash
npm run dev
```

### 3. 测试工作流

```bash
node test-local-workflow.js
```

### 4. 前端调用

前端无需修改，API 接口保持不变：

```typescript
const response = await fetch('/api/audit', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ input: '用户输入' })
});

const result = await response.json();
// result.score, result.decision, result.thought_chain, ...
```

## 扩展开发

### 添加新案例

编辑 `src/lib/fraudCaseDatabase.ts`：

```typescript
{
  id: 'FRAUD-2026-005',
  name: '新案例名称',
  attackMethod: '攻击手段',
  keywords: ['关键词1', '关键词2'],
  patterns: ['特征1', '特征2'],
  baseScore: 80,
  decision: 'WARN',
  description: '案例描述'
}
```

### 调整决策规则

编辑 `src/lib/workflowEngine.ts`：

```typescript
// 修改分支阈值
if (score >= 85) → 高风险  // 原来是 80
else if (score >= 65) → 中风险  // 原来是 60

// 修改决策规则
if (score >= 95) → HALT  // 原来是 90
```

### 添加新关键词

编辑 `src/lib/workflowEngine.ts`：

```typescript
const KEYWORD_DATABASE = {
  CRITICAL: [..., '新高风险词'],

  MEDIUM: [..., '新低风险词']

```

## 故障排查

### Q1: 工作流不生效

**检查**：
- `.env.local` 中 `USE_LOCAL_WORKFLOW=true`
- 开发服务器已重启
- 浏览器缓存已清除

### Q2: 评分不准确

**检查**：
- 关键词库是否完整
- 案例库是否匹配
- 权重计算是否合理

### Q3: 案例匹配失败

**检查**：
- 案例关键词是否覆盖
- 输入文本是否包含关键词
- 匹配逻辑是否正确
