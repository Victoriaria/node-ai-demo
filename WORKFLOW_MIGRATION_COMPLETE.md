## 工作流架构

### 原 Coze Workflow

```
开始 → 问答 → 条件分支 → 大模型1/2 → 结束
  ↓      ↓        ↓          ↓        ↓
依赖   依赖    依赖       依赖     依赖
Coze   Coze    Coze       Coze     Coze
```

**问题**：
- ❌ 完全依赖 Coze API
- ❌ 响应慢（15-30秒）
- ❌ 不可控
- ❌ 有调用成本
- ❌ 调试困难

### 新本地工作流

```
开始 → 问答节点 → 条件分支 → 大模型分析 → 结束
  ↓        ↓          ↓           ↓         ↓
本地    关键词提取   风险分流   深度分析   生成报告
代码    案例匹配     智能路由   决策引擎   格式化
```

**优势**：
- ✅ 完全本地化
- ✅ 响应快（<100ms）
- ✅ 100% 可控
- ✅ 零成本
- ✅ 易于调试和扩展

## 核心功能

### 1. 关键词识别

**三级关键词库**：
- **高风险**（权重15）：JD、换汇、拆单、洗钱、诈骗等
- **中风险**（权重8）：跨境、大额、转账、异常等
- **低风险**（权重3）：支付、汇款、收款等

### 2. 案例库匹配

**4个典型案例**：
1. FRAUD-2026-001：拆单洗钱（评分92）
2. FRAUD-2026-002：JD任务型诈骗（评分85）
3. FRAUD-2026-003：汇率套利（评分75）
4. FRAUD-2026-004：账户接管（评分98）

### 3. 智能评分

**评分逻辑**：
```typescript
基础分 = 关键词权重总和 × 3
案例加成 = max(匹配案例的基础评分)
特殊规则 = 跨境+大额+转账 组合加分
最终评分 = min(基础分 + 加成, 100)
```

### 4. 决策规则

| 评分范围 | 风险等级 | 决策 | 说明 |
|---------|---------|------|------|
| >= 90 | CRITICAL | HALT | 立即拦截，人工复核 |
| 60-89 | HIGH | WARN | 警告，需人工审核 |
| < 60 | LOW | ALLOW | 允许交易 |
| null | UNDETERMINED | INFO_REQUIRED | 信息不足 |

## 使用方式

### 1. 环境配置

在 `.env.local` 中设置：

```env
# 启用本地工作流（推荐）
USE_LOCAL_WORKFLOW=true

# 关闭 Coze API
USE_MOCK_DATA=false
```

### 2. 启动服务

```bash
npm run dev
```

### 3. 测试验证

```bash
node test-local-workflow.js
```

### 4. 前端调用

前端代码无需修改，API 接口保持不变：

```typescript
const response = await fetch('/api/audit', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ input: '用户输入' })
});

const result = await response.json();
// result.score, result.decision, result.thought_chain
```
