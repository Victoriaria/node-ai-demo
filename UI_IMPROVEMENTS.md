# UI 改进说明

## 修复内容

### 问题
初始状态显示"服务异常"，容易让用户误以为系统有问题。

### 解决方案
将初始状态的显示文案改为更友好的提示：

#### 修改前 ❌
- 显示文字："服务异常" / "Service Error"
- 颜色：灰色
- 给人感觉：系统出错了

#### 修改后 ✅
- 显示文字："待审计" / "Pending"
- 颜色：蓝色（品牌色）
- 给人感觉：等待用户输入，系统正常

## 状态说明

### 1. 初始状态（riskScore = null）
- **显示**："待审计"
- **颜色**：蓝色 (#00f2ff)
- **含义**：系统就绪，等待用户输入

### 2. 低风险（0-49分）
- **显示**：具体分数（如 20）
- **颜色**：绿色
- **决策**：ALLOW（允许）

### 3. 中风险（50-79分）
- **显示**：具体分数（如 70）
- **颜色**：黄色
- **决策**：WARN（警告）

### 4. 高风险（80-100分）
- **显示**：具体分数（如 94）
- **颜色**：红色
- **决策**：HALT（拦截）

## 使用流程

1. **打开页面**
   - 看到"待审计"（蓝色）
   - 表示系统正常，等待输入

2. **输入消息**
   - 例如："你好"
   - 点击"发送"按钮

3. **查看结果**
   - 风险评分显示具体数字
   - 颜色根据风险等级变化
   - AI回复显示在聊天区
